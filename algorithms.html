Enhancement Focus

  • Compare learned behavior with a shortest-path baseline
  • Show measurable results instead of only a successful run
  • Explain trade-offs through direct output data
Original Artifact

What the project did

The original project trained an intelligent pirate agent to move through a maze and find the treasure. It showed that the agent could learn, but it did not give a strong comparison between the learned path and a more direct path.

Enhanced Artifact

What I improved

I added a separate analysis file that uses breadth-first search as a shortest-path baseline. This made it possible to compare the learned policy with a clear benchmark and produce results that are easier to explain.

Training Episodes

1200 episodes were used during the evaluation run.

Success Rate

The learned policy reached the treasure from all free starting cells.

Path Quality

The learned average path length stayed close to the optimal average path length.

Measured results

  • Success rate: 100%
  • Average learned steps: 16.16
  • Average optimal steps: 15.64
  • Losses during the run: 65

Skills demonstrated

  • Algorithm analysis through direct comparison
  • Use of queues, dictionaries, and grid-based state logic
  • Understanding trade-offs between learning and optimal routing
